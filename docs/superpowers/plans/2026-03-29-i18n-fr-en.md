# French / English i18n Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development or executing-plans to implement task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Centralize all player-facing copy behind an internationalisation layer with **English** and **French** locales, switchable in the UI and persisted across sessions.

**Architecture:** Use **i18next** with **react-i18next** (fits Vite + React 19, works outside React via `i18n.t()` for `gameLogic.ts` and hooks). Locale JSON files are grouped by **namespace** (`common`, `ui`, `game`, `campaign`, `tour`) to keep merges sane. Runtime **stores resolved strings** in `GameState.logs` and similar (already plain strings): at write time, call `i18n.t(...)` so saved games and the corkboard show the language active when the event occurred, unless you later decide to re-localize logs on display (optional follow-up).

**Tech Stack:** `i18next`, `react-i18next`; optional `i18next-browser-languagedetector` for first visit; TypeScript resource typings via `i18next` declaration merging or a small `resources.d.ts`.

---

## File structure (new)

| Path | Responsibility |
|------|----------------|
| `src/i18n/index.ts` | `i18n.init({ resources, lng, fallbackLng: 'en', interpolation, ns, defaultNS })`; export `i18n` |
| `src/i18n/locales/en/common.json` | Shared labels: buttons, difficulty, aria, errors |
| `src/i18n/locales/en/ui.json` | Component copy: TopBar, Landing, Corkboard chrome, Clipboard tabs, grid hints, pause overlay |
| `src/i18n/locales/en/game.json` | Greetings (`GREETINGS`, etc.), activity log lines, `gameLogic` messages, hook log templates |
| `src/i18n/locales/en/campaign.json` | Nested by night/path or flat keys: `night1.newspaper`, `fired.morale.headline`, character `clueText` / `consequenceDescription` keys |
| `src/i18n/locales/en/tour.json` | Tour step titles + tooltips |
| `src/i18n/locales/fr/*.json` | Same keys as `en`; French translations |
| `src/main.tsx` | Import `./i18n` before render; wrap app in `I18nextProvider` if using v15+ pattern, or rely on `initReactI18next` + default import side effect |

**Note:** `campaign.json` can stay thin if you prefer **IDs in `campaignConfig.ts`** that reference keys like `campaign.night1.bodyLeft` instead of embedding English in TS — same for `firedConfig` and `characterRoster`.

---

## Library choice (why not X)

- **i18next + react-i18next:** Mature; **non-React** `i18n.t()` for `src/logic/gameLogic.ts` and hooks; pluralization and `{{name}}` interpolation; namespaces; no extra Vite plugin.
- **Lingui:** Excellent extraction workflow; needs build integration; heavier migration for a first i18n pass.
- **react-intl (FormatJS):** Fine, but `gameLogic` would use `intl.formatMessage` only if you pass formatters in — more plumbing.

Recommendation: **start with i18next**.

## Persistence and detection

- [ ] **Step 1:** On language change, `localStorage.setItem('i18n_lang', lng)` and `document.documentElement.lang = lng`.
- [ ] **Step 2:** On `i18n.init`, read `localStorage` first, then optional `navigator.language` prefix match (`fr` → `fr`), fallback `en`.

## Testing strategy

- [ ] **Step 1:** Add `src/i18n/__tests__/i18n.test.ts`: after importing configured `i18n`, assert `i18n.t('some.key', { lng: 'fr' })` equals expected French for a smoke set of keys.
- [ ] **Step 2:** In Vitest `setup` (or per file), ensure `i18n` is initialized before game logic tests that assert log text — or change those tests to assert **key presence / structure** instead of full English sentences.
- [ ] **Step 3:** Component test (optional): render `LandingPage` with `fr` and assert visible French for one button label.

## Risks and mitigations

| Risk | Mitigation |
|------|------------|
| Bundle size (large JSON) | Namespaces + lazy `import()` per namespace if needed later |
| Tests break on string equality | Central test `setupFiles` initializes `i18n` with `en` resources |
| Saved games / logs in “wrong” language | Accept snapshot language at event time, or add `displayLogKey` + translate in `ActivityLog` (larger refactor) |
| Duplicate keys | ESLint plugin `eslint-plugin-i18n-json` or custom script (optional follow-up) |
